Think of **sliding window** the same way you remember recursion.

For recursion, you remember:

1. **When to stop?** (base case)
2. **If not stopped, call itself again.**

For **sliding window**, remember these 3 questions:

# Sliding Window

### 1. Where does my window start?

```js
left = 0
right = 0
```

Imagine you're looking through a moving window on an array.

```
[7,1,5,3,6,4]
 ^
left,right
```

---

### 2. Should I expand or shrink the window?

Move the right pointer to include more elements.

```js
right++
```

Sometimes you also move the left pointer when the window becomes invalid.

**Expand → see more elements.**

**Shrink → throw away useless elements.**

---

### 3. Update the answer while moving

Whenever you find something better:

```js
answer = max(answer, currentWindowValue)
```

---

# Your Problem (Best Time to Buy and Sell Stock)

Array:

```js
[7,1,5,3,6,4]
```

### Step 1

Window starts:

```
7
^
L,R
```

Minimum price = 7

Profit = 0

---

### Step 2

Move right

```
7 1
^ ^
L R
```

1 is smaller than 7.

Buying at 7 is stupid.

Shift the left pointer:

```
7 1
  ^ ^
  L R
```

Now:

```
min = 1
```

---

### Step 3

Move right again

```
1 5
^ ^
L R
```

Profit:

```
5-1 = 4
```

Update answer:

```
profit = 4
```

---

### Step 4

Move right

```
1 5 3
^   ^
L   R
```

Profit:

```
3-1 = 2
```

Not better than 4.

Keep answer = 4.

---

### Step 5

Move right

```
1 5 3 6
^     ^
L     R
```

Profit:

```
6-1 = 5
```

Better!

```
profit = 5
```

---

### Step 6

Move right

```
1 5 3 6 4
^       ^
L       R
```

Profit:

```
4-1 = 3
```

Not better.

End.

Answer = 5.

---

# In layman's terms

Imagine you want to buy vegetables cheaply and sell them later at maximum profit.

* **Left pointer = buying day**
* **Right pointer = selling day**

As you walk day by day:

### If today's price is lower

Forget the previous buying day.

Move the buying day to today.

```js
if (prices[right] < prices[left])
    left = right;
```

### Otherwise

See how much profit you can make.

```js
profit = prices[right] - prices[left]
```

If profit is better than previous best:

```js
maxProfit = profit
```

---

# The memory trick

For recursion:

> Stop → Else call again.

For sliding window:

> **Start a window → Move right → If window becomes bad, move left → Update answer.**

---

## Fixed-size Sliding Window

Questions like:

> Maximum sum of 3 consecutive numbers

Remember:

1. Start window.
2. Expand right.
3. When size exceeds K, remove left.
4. Update answer.

---

## Variable-size Sliding Window

Questions like:

> Longest substring without repeating characters

Remember:

1. Start window.
2. Expand right.
3. If window becomes invalid, shrink left.
4. Update answer.

---

## Your stock problem is actually:

> Expand right day by day → if a better buying day appears, move left → update maximum profit.

So you can memorize sliding window as:

> **Expand → Shrink if needed → Update answer**

Exactly like recursion is:

> **Stop → Otherwise recurse**.

These two sentences are enough to solve most sliding window problems.
